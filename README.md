# 鸿蒙智能小车项目 (HarmonyOS Smart Car) README

本项目是一个基于鸿蒙系统 (LiteOS-M) 开发的智能小车项目，工程静态库名称为 `robot_car`[cite: 3]。系统集成超声波避障、红外循迹、OLED 状态显示、实体按键调速以及基于 Wi-Fi 和 MQTT 的远程遥控功能[cite: 1, 2, 4, 6, 7]。

下面是该项目的详细技术规格与使用说明。

## ⚙️ 硬件与引脚映射分配

项目对外设进行了精确的 GPIO 硬件资源分配，硬件连接说明如下：

* **直流电机 (PWM 控制)**
  * 左轮 A 相连接至 GPIO0，复用为 PWM3 输出[cite: 5]。
  * 左轮 B 相连接至 GPIO1，复用为 PWM4 输出[cite: 5]。
  * 右轮 A 相连接至 GPIO9，复用为 PWM0 输出[cite: 5]。
  * 右轮 B 相连接至 GPIO10，复用为 PWM1 输出[cite: 5]。
  * PWM 周期设定为 40000，输出频率为 4kHz，完美适配电机调速[cite: 5]。
* **超声波测距传感器**
  * 发送触发信号 (Trig) 的输出引脚连接 GPIO_7[cite: 1]。
  * 接收回波信号 (Echo) 的输入引脚连接 GPIO_8[cite: 1]。
* **红外循迹传感器 (TCRT5000)**
  * 左侧红外传感器连接至 GPIO_11，配置为输入模式[cite: 1]。
  * 右侧红外传感器连接至 GPIO_12，配置为输入模式[cite: 1]。
* **ADC 按键控制模块**
  * 使用单个模拟引脚读取多个按键状态，连接在 GPIO_5 上，使用 ADC 通道 2[cite: 2]。
* **OLED 显示屏**
  * 采用 I2C0 通信协议，波特率配置为 400kHz[cite: 4]。
  * SDA 引脚连接至 GPIO_13[cite: 4]。
  * SCL 引脚连接至 GPIO_14[cite: 4]。

## 🚀 核心工作模式与控制逻辑

系统的主任务 (`RobotTask`) 以超高频 (2ms 延迟) 运行，具备三种核心工作模式：停止 (MODE_STOP)、循迹 (MODE_TRACK) 与 遥控 (MODE_REMOTE)[cite: 7]。按下 S3 按键即可在这三种模式之间轮流切换[cite: 7]。

### 1. 自动循迹模式 (MODE_TRACK)
* **安全避障检测**：测距函数 `GetDistance` 循环获取前方距离，如果距离大于 0 且小于 15.0 厘米，小车将强制停止并触发 "Obstacle!" 动作[cite: 1, 7]。
* **智能微调转向**：当单侧传感器压到黑线时（左压线或右压线），小车会执行软转弯 (Soft L / Soft R)[cite: 7]。
* **终点智能识别**：当左右红外传感器同时检测到黑线（双黑）时，系统会启动双黑计数器[cite: 7]。当连续检测超过动态阈值（由直行速度计算得来）后，系统会确认到达终点 ("FINISH!") 并锁死车轮，自动切换至停止模式[cite: 7]。
* **丢线重寻机制 (坦克掉头)**：如果传感器完全脱离黑线（双白状态），双黑嫌疑会被解除，进入丢线防抖计数阶段[cite: 7]。如果在防抖时间以及微调周期内未能找到黑线，系统将触发极限转向阶段，根据记忆的最后一次转向记录执行原地的“坦克掉头”(SPIN L! / SPIN R!)[cite: 7]。

### 2. 远程遥控模式 (MODE_REMOTE)
* **全向移动支持**：在遥控模式下，小车会解析 MQTT 模块传入的遥控指令[cite: 6, 7]。
* **差速转向控制**：通过降低内侧车轮速度 (降至 0.75 倍) 甚至部分时间让内侧车轮倒退，实现完美的差速转弯[cite: 5]。
* **动作映射**：支持前向直行、后退、左转、右转、左前行驶、右前行驶、左后行驶、右后行驶以及原位待机等指令[cite: 7]。

### 3. ADC 按键与调速系统
实体按键根据电压区间判断按键事件，连续采样 64 次取平均值来有效防抖[cite: 2]。
* **S1 按键 (0.4V - 0.6V)**：触发减速指令，每次将直行速度减少 5（下限 50），转弯速度减少 1（下限 55）[cite: 2, 7]。
* **S2 按键 (0.8V - 1.1V)**：触发加速指令，每次将直行速度增加 5（上限 100），转弯速度增加 1（上限 65）[cite: 2, 7]。
* **S3 按键 (0.01V - 0.3V)**：触发工作模式切换[cite: 2, 7]。
注：也可通过 MQTT 发送 "S1"、"S2"、"S3" 字符串实现与实体按键一致的远程调速与切换[cite: 6, 7]。

## 🌐 网络与 MQTT 云端接入

小车内置了 Wi-Fi 模块与 Paho MQTT 客户端，系统启动后自动分配独立任务进行连接[cite: 3, 6]。

* **Wi-Fi 连接参数**：
  * 热点名称 (SSID)：`TJXsnhh666`[cite: 6]。
  * 密码：`12345678`，安全模式为 WPA2PSK[cite: 6]。
* **MQTT 服务器参数**：
  * 服务器 IP 地址：`192.168.230.35`，端口为 `1883`[cite: 6]。
  * 客户端 ID (ClientID)：`ohos_hi3861_car`[cite: 6]。
  * 鉴权用户名：`TJXXX`[cite: 6]。
  * 鉴权密码：`051211llq`[cite: 6]。
* **通信 Topic 与功能**：
  * **指令下发主题 (`ohossub`)**：小车订阅该主题。可接收调速指令 ("S1", "S2", "S3") 与方向控制指令 ("F", "B", "L", "R", "FL", "FR", "BL", "BR", "STOP")[cite: 6]。
  * **状态上报主题 (`car/status`)**：小车发布当前模式、执行动作以及直行、转弯速度到此主题（格式如 "模式,动作,直行速度,转弯速度"）[cite: 4, 6]。
  * **心跳保活主题 (`ID`)**：小车每秒循环向此主题发布 "Car Online" 字符串消息[cite: 6]。

## 💻 屏幕显示 (OLED UI 逻辑)

* **UI 布局**：屏幕外围有全局白色边框，内部划分为状态区和参数区[cite: 4]。
  * **第一行**：显示当前模式，如 `MODE: TRACKING` 等[cite: 4]。
  * **第二行**：显示当前运动状态，如 `Act : Soft L`[cite: 4]。
  * **下半区**：绘制区域分割线，实时显示直行速度 (`Spd`) 和转弯速度 (`Trn`) 参数值，并附带动态进度条（进度条宽度最大为 60 像素）[cite: 4]。
* **智能锁帧器**：为防止过高频率刷新导致的卡顿与硬件压力，屏幕刷新逻辑被设计为积累 5 次循环（约等于 10-20 FPS）并且当状态发生改变时才会重绘[cite: 7]。

## 🛠️ 项目构建与依赖说明

本项目通过鸿蒙的 `BUILD.gn` 文件进行工程配置与静态库构建[cite: 3]。
* **项目源文件包含**：`robot_core.c`, `motor_control.c`, `sensor.c`, `display.c`, `adc_key.c`, `mqtt_module.c`[cite: 3]。
* **外部库依赖**：项目依赖了 `ssd1306:oled_ssd1306` 用于屏幕驱动，以及 `pahomqtt:pahomqtt_static` 用于提供网络 MQTT 支持[cite: 3]。
* **编译头文件路径包含**：包含了鸿蒙底层的 `lite/include`，CMSIS 硬件抽象层接口，IoT 外设接口库以及 Paho MQTT 库相关路径[cite: 3]。
